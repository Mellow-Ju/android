# 투자 모니터링 앱 — UX 문서

> 버전: v0.1 | 작성일: 2026-05-13 | 상태: 검토 중

---

## 1. 앱 개요

### 목적
Google Sheets 기반 주식 관리 시스템과 연동하여, 모바일에서 언제 어디서나 투자 현황을 확인하고 거래를 기록하는 투자 모니터링 앱.

### 핵심 가치
- **빠른 기록**: 매수/매도 발생 즉시 모바일에서 입력
- **즉시 확인**: 전체 포트폴리오 현황을 한눈에
- **계좌별 관리**: ISA, IRP, 연금저축, 일반계좌 분리 확인

### MVP 범위 (1차 개발)
1. 종목 관리 (종목 추가/삭제/조회)
2. 매수/매도 히스토리 기록 및 조회
3. 전체 보유 종목 Dashboard
4. 포트폴리오 Dashboard

## 2. 사용자 여정 (User Journey)

### 시나리오 A: 신규 매수 기록
```
앱 실행 → 하단 탭 "거래" → FAB "+" 탭 → 매수/매도 입력 폼
→ [종목 선택 → 날짜 → 거래유형(매수) → 가격 → 수량 → 계좌 → 증권사]
→ 저장 → 거래 목록으로 복귀
```

### 시나리오 B: 포트폴리오 현황 확인
```
앱 실행 → 홈 탭 → 총 자산 요약 카드 확인
→ 종목별 리스트 스크롤 → 특정 종목 탭 → 종목 상세 (보유수량/평단/수익률)
```

### 시나리오 C: 계좌별 현황 확인
```
포트폴리오 탭 → 상단 계좌 탭 [전체 / ISA / IRP / 연금저축 / 일반계좌]
→ 해당 계좌 종목 리스트 + 비중 차트 확인
```

### 시나리오 D: 신규 종목 등록
```
종목 탭 → "+" FAB → 종목 추가 폼 [종목명 / 해외·국내 / Ticker]
→ 저장 → 종목 리스트로 복귀
```

## 5. 인터랙션 정의

### 5.1 거래 내역 리스트
- **스와이프 우→**: 삭제 확인 Dialog
- **탭**: 거래 상세 보기 (날짜/종목/계좌/증권사/가격/수량)
- **롱프레스**: 편집 모드 진입

### 5.2 종목 리스트
- **탭**: 해당 종목의 전체 거래 이력 화면
- **롱프레스**: 편집 / 삭제 옵션 BottomSheet

### 5.3 Dashboard 종목 카드
- **탭**: 종목 상세 (보유수량 / 평단 / 투자금 / 현재가 / 수익률 / 거래 이력)
- **Pull-to-refresh**: 현재가 갱신 (네트워크 호출)

### 5.4 필터 (거래 탭)
- 기간 칩: 1주 / 1달 / 3달 / 전체 (기본: 전체)
- 계좌 칩: 드롭다운 멀티 선택
- 거래유형 칩: 매수 / 매도 / 전체 (토글)

## 6. 데이터 모델 (UI 관점)

### Stock (종목)
| 필드 | 타입 | 비고 |
|------|------|------|
| id | Long | PK (자동증가) |
| name | String | 종목명 |
| marketType | Enum | 해외 / 국내 |
| ticker | String | SCHD, 005930 등 |

### Trade (거래)
| 필드 | 타입 | 비고 |
|------|------|------|
| id | Long | PK |
| date | LocalDate | 거래일 |
| stockId | Long | FK → Stock |
| tradeType | Enum | 매수 / 매도 |
| price | Double | 가격 |
| quantity | Double | 수량 |
| totalAmount | Double | price × quantity (계산값) |
| accountType | Enum | 연금저축 / ISA / IRP / 일반계좌 / 연금저축2 |
| broker | Enum | 삼성 / 토스 / NH / 신한 / 이베스트 |

### HoldingSummary (보유 현황 — 계산값)
| 필드 | 타입 | 비고 |
|------|------|------|
| stock | Stock | |
| totalQuantity | Double | 총 보유수량 |
| avgPrice | Double | 평균 단가 (이동평균법) |
| totalInvestment | Double | 총 투자금 |

## 7. 엣지 케이스 & 예외 처리

| 상황 | 처리 방법 |
|------|----------|
| 현재가 로딩 실패 | "-" 표시 + 마지막 갱신 시각 표시 |
| 매도 수량 > 보유 수량 | 입력 폼에서 경고 메시지 + 저장 불가 |
| 종목 없이 거래 입력 | 종목 추가 유도 Dialog |
| 전체 보유수량 = 0 (완전 매도) | Dashboard에서 해당 종목 숨김 (설정으로 on/off) |
| 네트워크 없음 | 오프라인 모드 (로컬 DB 데이터만 표시, 현재가 미표시) |
| 환율 가져오기 실패 | 마지막 저장 환율 사용, 기본값 1,350 |

## 8. 접근성 & 로컬라이제이션

- 최소 텍스트 크기: 14sp (Material 3 기준)
- 수익(▲): 파란색 / 손실(▼): 빨간색 — 한국 금융 관례 적용
- 숫자 포맷: 국내 원화 `#,##0` / 해외 달러 `$#,##0.00`
- 날짜 포맷: `yyyy.MM.dd`
- 언어: 한국어 (ko-KR) 기본, 영어 확장 고려

## 10. 데이터 가져오기 (Import)

### 10.1 엑셀 파일 Import (매수/매도 데이터)

기존 Google Sheets에 쌓인 매수/매도 이력을 앱으로 가져오는 기능.

**사용자 플로우**
```
설정 탭 (또는 거래 탭 상단 메뉴) → "엑셀 파일로 가져오기"
→ 파일 선택 (xlsx / xls) → 컬럼 매핑 미리보기 화면
→ 데이터 확인 → "가져오기" → 완료 (성공/실패 건수 표시)
```

**지원 파일 형식**
- `.xlsx` (Excel 2007 이상)
- `.xls` (Excel 97–2003)
- Google Sheets에서 다운로드한 xlsx

**컬럼 매핑 규칙** (Google Sheets "매수/매도" 시트 기준)

| 엑셀 컬럼 | 앱 필드 | 비고 |
|-----------|---------|------|
| 년도, 월, 일 | date | 3컬럼 합산 → LocalDate |
| 종목명 | stock.name | 미등록 종목 자동 추가 |
| 매수/매도 가격 | price | $ 기호 제거 후 Double 변환 |
| 주식 수 | quantity | |
| 총 금액 | totalAmount | 없으면 price × quantity 계산 |
| 해외/국내 | stock.marketType | |
| 거래유형 | tradeType | 매수 / 매도 |
| 매수/매도 계좌 | accountType | |
| 증권사 | broker | |

**예외 처리**
- 필수 컬럼 누락 행: 해당 행 스킵 + 오류 목록 표시
- 미등록 종목명: 종목 자동 생성 (Ticker 미입력 상태로 저장, 이후 수동 보완)
- 중복 데이터: 날짜+종목+가격+수량 동일 시 중복 경고 → 사용자 선택 (덮어쓰기 / 스킵)

**UI 스케치**
```
┌─────────────────────────────────┐
│ 엑셀 파일 가져오기               │
├─────────────────────────────────┤
│  [파일 선택 또는 여기에 끌어놓기] │
│                                 │
│  ─── 미리보기 (상위 5행) ─────── │
│  날짜     종목  유형  수량  가격  │
│  2024.01  SCHD  매수  50   27.1  │
│  2024.02  QQQ   매수  10  420.0  │
│  ...                            │
│                                 │
│  총 156건 / 오류 2건            │
│  ⚠ 오류 목록 보기               │
│                                 │
│  [취소]           [가져오기]     │
└─────────────────────────────────┘
```

## 11. 현재가 API 전략

현재가 조회는 단계적 폴백(fallback) 방식으로 구현.

```
1순위: Google Finance (GOOGLEFINANCE 함수 방식 — Android 직접 호출 불가)
       → 대신 비공식 엔드포인트 시도
2순위: Yahoo Finance API (yfinance 호환 엔드포인트)
       — 해외 종목에 강점, 무료 사용 가능
3순위: Alpha Vantage API
       — 무료 티어: 25 req/day, API Key 필요
       — 국내 종목 지원 제한적
국내 종목 보완: 네이버 금융 크롤링 또는 한국투자증권 Open API 검토
```

**구현 방침**
- Repository 패턴으로 API 소스를 추상화 → 폴백 전환 시 UI 영향 없음
- 캐시: 마지막 성공한 현재가를 Room DB에 저장, 오프라인 시 표시
- 갱신 주기: 앱 포그라운드 진입 시 + Pull-to-refresh 시
